# MemWise

Windows 智能内存看护工具。**不杀进程、不挂起线程、不注入、不联网。** 纯 ctypes Win32 API，零外部依赖。

---

## 用一分钟看懂它能做什么

打开 `MemWise.exe`，点击「守护」——然后就够了。它在系统托盘里安静运行，当内存压力上升时自动清理系统缓存和进程闲置页，压力回落时自动降低频率。全程不需要你操作任何选项。

你也可以点「⚡ 优化」立刻跑一轮，或者按 `Ctrl+Shift+M` 热键快速调出窗口。

## 三点让它和其他内存工具不一样

### 每个进程都有独立的学习模型

不是简单的「WS 大就清」。每个进程维护一个 Beta(α,β) 先验分布——清理成功 α+1，失败 β+1。Thompson Sampling 从分布中采样得到 θ，决定这个进程值不值得清。同一个浏览器标签页，之前清过且 PF 反馈好 → θ 高 → 优先清；清过但导致页面调入卡顿 → θ 低 → 暂时跳过。

除了 α/β，还追踪 5 维特征（WS、波动率、PF 成本、置信度、增长斜率）做上下文修正，以及双 EWMA（快慢速）识别收益加速信号。

### 有一套完整的效率反馈系统（EFIS）

这不是一个静态规则引擎。每隔 30 tick（约 15 分钟），EFIS 会评估过去一段时间的清理效率——如果某个调整（比如降低了 θ 门槛）导致效率下降，**自动回滚**到之前的参数；如果效果好，记录到该场景的历史最优参数中，下次进入同一场景（游戏/开发/浏览/通用）直接沿用。

它有实验日志、场景记忆、窗口化评估（排除用户操作导致的混杂因素）、相对步长（大参数大步调小参数小步调）。这套系统在持续自我调参。

### PID 控制器决定「现在该不该清」

一个连续反馈 PID 控制器盯着内存使用率：比例项响应即时压力，积分项消除稳态偏差，微分项抑制震荡。目标 45%，当内存超过 55% 时清理强度开始线性上升，超过 75% 时进入高压模式（tick 间隔从 30s 缩到 10s）。

普通工具只有「已满/未满」两种状态。这是一个连续调节的闭环。

## 架构概览

```
用户界面 (tkinter)
    │
    ├── 守护模式 (独立线程)
    │   ├── Sniffer 进程快照
    │   ├── Learner 学习引擎
    │   │   ├── Thompson Sampling (α/β)
    │   │   ├── 上下文修正 (sign-based + 批量梯度)
    │   │   ├── 双 EWMA 加速信号
    │   │   ├── 泄漏检测 (双阈值 Z>2.0 / 1.5)
    │   │   ├── 预判式清理 (增长斜率)
    │   │   └── 探索奖励 (好奇心+不确定性)
    │   ├── Judger PID 控制器
    │   ├── Cleaner 3 层清理引擎
    │   │   ├── Layer1: 7 种系统操作
    │   │   ├── Layer2: 进程排序+Probe+Trim
    │   │   └── Layer3: 深度重复
    │   ├── EFIS 效率反馈
    │   │   ├── 实验日志+自动回滚
    │   │   ├── 场景参数记忆
    │   │   ├── 窗口化评估
    │   │   └── 历史最优参数回归
    │   └── 动态 tick (10~60s)
    │
    ├── 手动优化 (3轮深度+累计释放)
    └── 日志系统 (线程安全消息队列)
```

## 快速开始

```bash
pip install pyyaml          # 可选
python memwise_gui.py       # 启动图形界面
python memwise.py daemon    # CLI 守护模式
```

或者直接从 [Releases](https://github.com/fuguangbeta/MemWise/releases) 下载 exe。

## 命令行

```
memwise.py status             查看内存状态
memwise.py learn [分钟]       学习进程行为
memwise.py optimize [--mode]  一键优化 (quick/normal/deep/full)
memwise.py daemon             守护模式
memwise.py profile <PID>      查看进程画像
```

## 系统要求

- Windows 10 20H1+ / Windows 11
- 管理员权限（缓存清理需要；进程 EmptyWorkingSet 不受限）
