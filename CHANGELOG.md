# MemWise 更新日志

> **MemWise** — Windows 智能内存看护工具。不杀进程、不挂起线程、不注入、不联网。
> 纯 ctypes Win32 API，零外部依赖。

---

## v1.2 (2026年6月 — 当前)

> 迄今为止最大更新，涵盖学习系统、算法重构、大量 Bug 修复与 UI 改进。

| 优先级 | 问题 | 解决方案 |
|:---:|------|----------|
| **P0** | Thompson θ 双定义导致上下文增强完全失效 | 删除重复 `θ` 定义，5 维上下文修正生效 |

| **P1** | PID 控制器在 daemon 内外各更新一次 → 控制量翻倍 | `optimize()` 接受 `aggressiveness` 参数统一入口 |
| **P1** | 内存优先级映射反向（高 θ 进程拿到最高优先级，反而被优先回收） | 低 θ 进程跳过不设，高 θ 进程设为 `VERY_LOW`（正确方向） |
| **P1** | `can_probe` 不跳过前台进程 | 新增前台进程检查 |
| **P2** | `can_probe` 无 θ 过滤 | 样本数 ≥ 5 且 θ < 0.2 时跳过 |
| **P2** | `purge_expired` 冷却时间 +3600 冗余 | 改为 `now`，立即生效 |
| **P2** | 守护模式下 CLI 与 GUI 清理路径不统一 | 统一走 `cleaner.optimize()` |
| **P2** | 内存优先级永不刷新 | 每 30 tick 重新评估优先级 |
| **P3** | 测量用快照使用旧数据 | 改为实时读取工作集 (WS) |

| — | 残留 `.tmp` 临时文件 | 清理 stale 临时文件 |

### 学习系统 (Learning System)

| 变更 | 说明 |
|------|------|
| **上下文模型从 Probe 学习** | `record_probe` 增加 `_update_ctx_weights`，使上下文权重也参与反馈 |
| **置信度改用 Beta 分布标准差** | 替代简单计数，置信度 = `β / (α + β)`，更准确反映不确定性 |
| **Beta 分布时间衰减** | 每次 `feed` 对 `(α, β)` 乘以衰减因子，逐渐遗忘旧数据，适应新行为 |
| **自适应学习率** | 置信度高时微调（小步），置信度低时大胆探索（大步） |
| **失败计数逐渐遗忘** | 不再硬重置失败计数，改用衰减曲线 |

### 算法改进 (Algorithm)

| 变更 | 说明 |
|------|------|
| **复合评分排序** | `θ × (0.5 + 0.3 × 置信度 + 0.2 × ROI)`，综合考虑潜力、信心与性价比 |
| **自适应清理轮数** | WS < 50MB → 2 轮；50~200MB → 3 轮；> 200MB → 4 轮 |
| **Standby 频率门控** | 最小间隔 60 秒；可用内存 > 25% 跳过重型清理 |
| **配置热加载** | 守护模式每 60 秒检查 `config.yaml`，修改即时生效 |
| **全新全屏游戏检测** | `is_foreground_fullscreen` 检测全屏窗口，准确识别游戏运行状态 |
| **游戏名单扩充** | 新增国产 / Steam / UWP / Xbox 平台 30+ 条游戏进程名 |
| **进程内存排行榜** | GUI 新增按钮，树状显示进程内存占用排行 |
| **单击终止进程** | 排行榜中单击进程行弹窗确认后即可终止 |
| **鼠标悬浮高亮** | 排行榜行悬浮高亮，提升浏览体验 |
| **默认清理模式** | 由 `quick` 改为 `normal`，更均衡的清理策略 |
| **state.json 过滤收紧** | 记录保留从 30 天 / 10 样本 → 7 天 / 5 样本，更快淘汰旧数据 |

### 自启动优化

| 变更 | 说明 |
|------|------|
| **静默自启动** | 新增 `--minimized` 参数，窗口创建即 `withdraw`，全程无闪烁 |
| **自启动即守护** | 自启动时立即进入守护模式（不再等 300ms 启动延迟） |
| **统一追加 --minimized** | 所有自启动路径（CLI + GUI）均追加 `--minimized` 参数 |

## v1.1 (2026年初)

### 主要新功能

| 功能 | 说明 |
|------|------|
| **游戏模式** | 内置 80+ 游戏名单，运行时自动调整后台进程内存优先级 |
| **双次清理** | 每次清理执行两次 `EmptyWorkingSet`，自适应间隔 300ms |
| **配置统一** | 迁移至 `core/config.py`，统一管理所有配置项 |
| **θ 缓存** | Thompson Sampling 参数缓存，避免重复计算 |
| **画像自动清理** | 超过 30 天的进程画像自动清除 |
| **四档颜色图标** | 系统托盘图标根据内存压力变色 |
| **守护模式完整分层清理** | 支持完整分层清理策略（L1~L3） |
| **操作过滤** | 智能过滤低价值操作，减少无用功 |
| **30 进程上限** | 单轮清理最多 30 个进程，防止串行堆积 |
| **GUI 居中 / 弹窗优化** | 窗口居中显示，弹窗体验改进 |

### 基础框架

- 引入 Thompson Sampling 探针机制（Probe）
- PID 控制器动态调节清理强度
- 守护模式后台持续运行（每 30 秒采集快照）
- CLI 交互界面（`status` / `learn` / `optimize` / `daemon` / `profile`）
- 系统托盘 + 全局热键（`Ctrl+Shift+M`）
