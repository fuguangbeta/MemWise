# MemWise 更新日志

> **MemWise** — Windows 智能内存看护工具。不杀进程、不挂起线程、不注入、不联网。
> 纯 ctypes Win32 API，零外部依赖。

## v2.2 (2026年7月)

> 本轮的核心主线是两条：效率评分体系从"释放量跟踪器"彻底蜕变为"算法效能仪表盘"，以及清理模式从"词不达意的四档"重塑为真正有操作梯度的四级分层。同步完成了日志清屏逻辑的确定性重建、工具提示的全面审计更正，以及自推出以来累积的大量细节修复。

### ERIS 效率评分彻底重塑（含游戏模式极端异常值平滑）

效率评分经历了从 v3 到 v5 的三次迭代，最终形成为一套以 EWMA 自适应基线为核心的仪表盘体系。

v3 的五维采用固定阈值归一化，在实际运行中迅速暴露出结构性缺陷——预测精准度和学习收敛度长期停留在地板值，副作用控制与探索完备度则迅速饱和至满分。几何平均的数学特性使得仅有两维触底就将全局评分压低至不变的水平线，连续五十轮恒定于 42%，完全丧失了波动性和区分度。

v4 尝试引入 EWMA 基线归一化替代固定阈值，使得每个维度的评分基准从硬编码常数变为自身历史均值。这个方向正确但实现存在两处偏差：前三轮的首轮异常高释放量直接锚定了 EWMA 基线于虚高位置，导致后续正常释放量全部被判定为"超标"；加权平均中波动维度权重仅 40%，被稳定维度的主导权重压制了波动幅度。

v5 完成了最终的收敛方案。五项维度重新定义为预测精准度、释放效率、副作用控制、参数稳定度和探索完备度，每维独立运行自身的 EWMA 衰减基线，本轮实际值除以基线得出相对比值，加权算术平均产出最终分数。游戏模式下启用的 soft sigmoid 压缩（ratio×3/(ratio+2)，渐近上限 3.0）和自适应 EWMA（λ=0.25+0.25×min(1,偏离度)）协同工作，使得游戏横扫导致的数百倍异常比值被压制到可读范围，同时基线快速加速收敛至新常态——首轮 ~170%、次轮 ~150%、3-5 轮内回归正常波动。，原始释放量不参与任何维度的构成，评分严格反映算法引擎相对于自身历史常态的运转质量。EWMA 基线以独立 JSON 文件在守护停止与关闭时持久化，重启后自动加载，无需重新收敛。

评分范围从硬性 0-100 扩展为 0-100%+，上不封顶。折线贴顶不溢出画布，超过 100% 的折点自动切换为金色以突出优秀表现，低于 35% 的折点切换为珊瑚红色以示警戒。前三轮收敛期折线及折点固定使用虚线描边并显示"影响因素分析中…"，第四轮起根据效率变化自动选取最优影响因子——上升时取贡献最大的正面因子，下降时取恶化最重的负面因子，持平则选取最偏离中性的维度。同维度的正负因子之间增加防振荡保护，不会在相邻轮次间来回跳变，引发用户困惑。当效率值超过 100% 或低于 35% 时，因子极性由整体水平强制决定——前者永远显示正面词条，后者永远显示负面词条——整体的优异或低迷压倒个别的维度波动。连续三轮同向变化时追加持续改善或持续下滑的趋势标签。

实现过程中修复了多项关键缺陷。守护线程与图表渲染线程之间的状态污染问题通过引入只读调用模式解决，EWMA 基线、因子趋势记录和上轮比较值均仅供守护线程写入。Beta 分布标准差计算中因极端衰减产生的复数被双保险机制拦截——源头 clamp α/β 至安全范围，排序前二次过滤实数子集。元认知校准的参与门槛从 clean_count≥5 降低至 ≥2，使新画像也能贡献校准信号并产生有意义的偏差反馈。EFIS 新增 `_last_adjust_cycle` 属性，为适应性维度提供真实的调参频率数据源。

### 清理模式梯度重塑

四种清理模式从"花式命名但操作雷同"彻底改造为真正由轻到重的四级分层。

快速模式仅执行三步无副作用系统操作——standby 清除、modified 页写回、注册表缓存清理——不触碰任何进程工作集，零卡顿，适合临时快速释放场景。标准模式在前者基础上增加全进程 trim（Thompson 联合 Kalman 排序选择）、系统级 WS 全清和卷缓存冲刷，但刻意跳过文件缓存操作，在"不产生副作用"的前提下达到优化极限。深度模式首次引入文件缓存清除和 standby 三层深度收割，释放量显著提升，代价是缓存重建期间文件操作短暂减速。全量模式在深度基础上追加进程回弹二轮追捕——对已修剪且工作集超过 10MB 的进程再次调用 EmptyWorkingSet，捕获短时间内重新分配的页面。

守护周期的 gap-fill 压制阶段同步受益于分层设计——从一个宽泛的"轻量"概念精确固定为四个操作（standby、low-priority standby、modified、registry），不再包括文件缓存和卷冲刷，既保持了高频压制的初衷又消除了磁盘 I/O 竞争。

`_layer1_memreduct` 新增一个以集合形式传入的操作过滤器，各模式仅需声明所需操作键名而无须维护多套独立管线函数。游戏模式的相关守卫（跳过 WS 全清、文件缓存、卷冲刷）与操作过滤器的交集自然保证双重保护。

### 日志清屏逻辑重建

旧版清屏依赖一个复杂的双层判断——基于文本控件可见行数与字体行高的比率，再叠加新周期事件标记。这一设计的核心问题在于周期末的消息到达顺序不可预测：前半部分消息已在旧日志下方显示，后半部分到达时触发行数阈值导致清屏，前半部分随之消失。

新方案采用二分法：所有属于周期末输出的消息——元认知校准、EFIS 诊断、概念漂移、探测试探、learn 和 cleaner 的暂存消息、以及最终的汇总行——在守护线程内全量缓存至一个列表中，周期末一次性打包发送。主线程收到批量消息后计算当前面板行数，若面板行数与批量总和超过七则清屏，随后逐条插入全部缓存消息。显示屏上本轮所有消息在空白画布上完整呈现，不受消息数量不可预测的影响。

按钮点击、游戏检测等实时交互产生的消息仍走即时通道，每次插入前执行相同的七行判断，超过则清旧再输出。旧版中与此相关的 `_log_should_clear`、`_clear_log`、`_new_cycle_event`、`log_flush` 处理器及已废弃的 `efis` 消息类型均被移除。

### 游戏模式交互完善

手动切换游戏模式新增自定义风格的确认弹窗，使用深色背景和 ttk 按钮，与程序整体视觉风格一致。窗口关闭时自动清理。

退出检测从需要连续两个探测周期均未发现游戏进程缩短为即时模式——游戏进程消失后在下一个探测周期立即恢复常规参数。Steam 平台进程从内置游戏名单中移除，避免非游戏场景下的误触发。

---

### 图表渲染细节

折点与折线的层叠顺序被显式设定为先折线后折点，确保折点圆完整覆盖在折线上方而非被线条穿透。悬浮提示框的尺寸从固定宽高改为基于 canvas 文字包围盒自适应计算，带趋势标签（如持续改善/持续下滑）的三行信息也能完整包裹。悬浮提示的触发位置从跟随鼠标改动为固定于对应柱条的水平中心，消除晃动感。前三个数据点的虚线描边判断从可见窗口长度改为全局绝对索引，前三轮永久虚线不受绘图缩放影响。
